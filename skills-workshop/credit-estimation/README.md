# Credit Estimation Toolkit

Measure, estimate, and plan your AWS Kiro credit usage for AI-powered test generation.

This toolkit helps you answer: **"How many Kiro credits will my team use this sprint?"**

## Quick Start

```bash
# 1. Initialize the results file
cd skills-workshop/credit-estimation
python track_credits.py init

# 2. Run a benchmark (interactive, measures your baseline cost)
python track_credits.py benchmark java-simple

# 3. Estimate credits for your real code
python track_credits.py estimate --type java-unit --source-lines 200 --mocks 3 --tests 3
```

## Prerequisites

- Python 3.10+
- pyyaml (`pip install pyyaml`)
- AWS Kiro CLI installed and authenticated (or access to [kiro.dev](https://kiro.dev))

## Commands

| Command | Description |
|---------|-------------|
| `init` | Initialize results file |
| `add <name> <start> <end> [notes]` | Record a test result |
| `benchmark <scenario>` | Run an interactive benchmark |
| `estimate --type <type> [options]` | Estimate credits for a workload |
| `plan [options]` | Estimate credits for a sprint |
| `baselines` | Show measured baselines |
| `summary [--period weekly]` | Show results summary |
| `export` | Export results to markdown |

Use `--output-dir DIR` with any command to specify a custom output directory.
